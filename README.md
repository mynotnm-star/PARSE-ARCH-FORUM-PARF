# Arch Linux Forum Multi-Threaded Parser

Швидкий та ефективний багатопоточний парсер офіційного форуму Arch Linux (bbs.archlinux.org), написаний на Python з використанням бібліотеки BeautifulSoup4 та модуля ThreadPoolExecutor.

## 🚀 Особливості

- **Багатопотоковість:** Одночасне виконання до 20 запитів завдяки `ThreadPoolExecutor`, що значно прискорює збір даних.
- **Маскування запитів:** Автоматична генерація випадкових заголовків `User-Agent` за допомогою бібліотеки `fake-useragent` для захисту від блокувань.
- **Оптимізація запису:** Потоки формують масиви даних локально і записують їх у файл пакетами, що мінімізує затримки вводу-виводу (I/O).
- **Безпека виконання:** Скрипт захищений від падінь у разі таймаутів або мережевих помилок окремих запитів.

## 🛠 Технологічний стек

- **Мова програмування:** Python 3.10+
- **Бібліотеки:** `requests`, `beautifulsoup4`, `lxml`, `fake-useragent`
- **Вбудовані модулі:** `concurrent.futures` (ThreadPoolExecutor), `time`

## 📦 Встановлення та налаштування

1. **Клонуйте репозиторій:**
   ```bash
   git clone https://github.com
   cd PARSE-ARCH-FORUM-PARF
   ```

2. **Створіть та активуйте віртуальне середовище:**
   ```bash
   python -m venv venv
   source venv/bin/activate  # Для Windows: venv\Scripts\activate
   ```

3. **Встановіть необхідні залежності:**
   Створіть файл `requirements.txt` (якщо його немає) та додайте туди:
   ```text
   requests
   beautifulsoup4
   lxml
   fake-useragent
   ```
   Після цього виконайте команду:
   ```bash
   pip install -r requirements.txt
   ```

## 💻 Використання

Запустіть головний скрипт для старту парсингу:
```bash
python main.py
```

### Логіка роботи:
- Скрипт генерує завдання для парсингу категорій форуму з ID від `1` до `50` та сторінок пагінації від `1` до `2766`.
- Дані обробляються у 20 паралельних потоків.

## 💾 Формат вихідних даних

Результати парсингу зберігаються у файл **`forumarch_parsing.txt`** у корінь проєкту. Кожен рядок має таку структуру:

```text
Форум ID:1 | Стр:1 | Тема: System freeze after latest kernel update | Автор: arch_user99 | Ответов: 14 | Просмотров: 342
Форум ID:1 | Стр:1 | Тема: How to install Arch on UEFI | Автор: linux_fan | Ответов: 5 | Просмотров: 1201
```
